# main.html 应用卡片列表改造计划

## 一、需求分析

根据用户提供的流程图，需要实现以下功能：

| 功能 | 描述 |
|------|------|
| 显示应用卡片列表 | 展示用户创建的应用卡片，使用 card.png 作为卡片图片 |
| 新建应用 | 点击新建按钮 → 弹出表单 → 输入名称和图片 → 创建卡片 |
| 编辑应用 | 点击卡片 → 进入三维场景编辑页面 |
| 删除应用 | 点击删除 → 确认对话框 → 删除或取消 |
| 复制应用 | 点击复制 → 输入新名称 → 创建副本卡片 |

## 五、交互流程

### 新建应用
1. 点击「新建应用」按钮
2. 弹出新建表单模态框
3. 输入应用名称
4. 选择应用图片（可选，默认使用 card.png）
5. 点击确定 → 创建新卡片 → 添加到列表
6. 点击取消 → 关闭模态框

### 编辑应用
1. 点击卡片的「编辑」按钮
2. 跳转到 3d-scene.html 页面

### 删除应用
1. 点击卡片的「删除」按钮
2. 弹出确认删除对话框
3. 点击「确认删除」→ 移除卡片
4. 点击「取消」→ 关闭对话框

### 复制应用
1. 点击卡片的「复制」按钮
2. 弹出复制表单模态框
3. 输入新应用名称
4. 点击确定 → 创建副本卡片
5. 点击取消 → 关闭模态框

## 六、风险与注意事项

1. **图片加载失败处理**：使用默认图片 card.png 作为备选
2. **数据持久化**：使用 localStorage 存储应用列表
3. **空列表状态**：当没有应用时显示提示信息
4. **响应式布局**：支持移动端显示
